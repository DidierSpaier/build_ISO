This batch of updates provides the LXQt 2.1.0 desktop, thanks for his great
work to Frank Honolka.

It also provides a rebuilt slapt-get allowing to remove obsolete packages:
Quoting "man slapt-get" after its update:
       --remove-obsolete
              Remove  installed  packages  that are no longer available.  This
              works for --dist-upgrade only. This removes packages that are no
              longer  available from any current package sources, but are cur‐
              rently installed.  Be careful as this will  also  remove  custom
              built  packages as well as packages coming from a repository not
              enabled in /etc/slapt-get/slapt-getrc.  Always use it  with  the
              -s option first to check what would be removed.
So to remove obsolete packages type as root:
slapt-get --dist-upgrade --remove-obsolete s
Carefully review the list of packages that would be reviewed before going
further. Be aware that if you enabled a repository, intsalles packages from it
then disabled it, the command above will propose to remove all packages from
this repository, if not available from another one.
If you agree with the list,  run the command above without the -s option or
remove only the packages you want to get rid of running for each of them
removepkg <package name>
or
spkg -d <package name>

About LXQt:
first run as usual and as root:
slapt-get -u
slapt-get --upgrade
dotnew (or dotnew-gtk)

If you want to get the LXQt desktop and associated applications, then type as
root or using sudo and preferably in a console (in text mode): 
slapt-get --install-set lxqt

Then to use it type as regular user:
session-chooser lxqt
Be aware that in the lightdm greeter the default session will still be mate
for now, so you will need to change it at time of log in. Alternatively use
login-chooser to start in text mode then type startx to start LXQt.
Caveat: LXQt is not as accessible as MATE for blind users.

If you have installed and used a previous version of LXQt, to update its
appearance type as regular user:
cd
rm -r .config/lxqt
rm -r .config/pcmanfmqt

xinit-1.4.2-x86_64-1slint.txz: Upgraded.
  This release contains a bug fix: now when running startx an xauth
  authorisation file is created in /tmp. This avoids displaying an error
  message like this one:
  xauth: file /home/didier/.serverauth.1090 does not exist
  A new script newstartx is also provided. Typing newstartx has the same
  effect as typing startx, with one exception: instead of connecting the
  graphical session to the first available console, it connects to the
  console where newstartx was run.
  For example, assuming a default /etc/fstab (6 consoles open at startup),
  typing newstartx from tty1 will connect the graphical sessions to tty1,
  instead of tty7.
  For some (still unknown to me) reason this also allows to change the
  time settings from the Clock applet of the Mate panel (via Preferences
  after right-clicking on the applet), just like when starting the Mate
  from lightdm. However, the time settings are instead disabled if the
  session was started by typing startx. I would be grateful for an
  explanation of this discrepancy - Didier.

# SOURCE=https://slackware.uk/gfs/15.0/41.10/x86_64/:OFFICIAL

SOURCE=https://slackware.uk/gfs/15.0/41.10/x86_64/:PREFERRED

slapt-get -u

slapt-get --install-set slint

https://slackware.uk/gfs/15.0/41.10/x86_64/
To know more, please read when available:
https://slackware.uk/gfs/15.0/41.10/x86_64/README
Short instructions:
to get the packages:
slapt-get --add-keys
slapt-get --update
slapt-get --install-set gnome
You can choose gdm as login manager in login-chooser.
You can choose gnome or gnome-classic in session-chooser.
To switch on/off (toggle) the orca screen reader in gdm press Alt+Super+S
To remove all packages of the gnome set type the two commands below:
cd /var/lib/pkgtools/packages
removepkg *gfs

Sun Oct 01 00:00:00 UTC 2023
News:
1. This batch of updates provides the ability to load early microcode
for Intel and AMD CPU, as stated for instance in:
https://www.kernel.org/doc/Documentation/x86/microcode.txt
https://wiki.archlinux.org/title/microcode
We will, use the method mentioned under 1.2.2.2 GRUB. the new file
/boot/grub/grub.cfg written by update-grub after a kernel upgrade will
include a line similar to this one:
 initrd	/@/boot/intel-ucode.img /@/boot/amd-ucode.img /@/boot/initramfs-6.1.55.img
This will load the iniramfs carrying the microcode for Intel and AMD CPUs early.
2. From now on we will avoid to ship in Slint packages already available in the
repository extra-15.0 provided by Geage Vlahavas, Salix maintainer:
including this line in /etc/slapt-get/slapt-getrc
#  SOURCE=https://slackware.uk/salix/x86_64/extra-15.0/

  Changes:
  1. New users will get in their ~/.profile file these lines:
export GROFF_ENCODING=UTF-8
export XDG_CACHE_HOME=/run/user/$(id -u)
export XDG_RUNTIME_HOME=/run/user/$(id -u)
chmod 700 /run/user/$(id -u)
mkdir -p $HOME/.ccache
  Existing users can just include their line in their ~/.profile (the first one
  should already exist).
  The goal is to write the files in ~/.cache in RAM, to avoid useless writes on
  disk.
  2. Set by default the session as "mate", to avoid that new users typing "startx"
  without having set the graphical session using the session-chooser utility just
  get a blank screen with an xterm.

Sint64-15.0beta3.iso is available in:
https://slackware.uk/slint/x86_64/slint-15.0/iso/
This ISO wheighs 4527839232, so should fit in a DVD.
In this aim I have removed big packages that can be installed later, or
are useless in Slint context, namely: mkinitrd (supersed by dracut),
elilo and lilo (superseded by GRUB), slackpkg (we use slapt-get),
slacktrack, seamonkey, accountsservice, geequie, rust, zuku-openjdk and
fluid-soundfont.
Now the choice of desktop made at end of installation will be effective
after a reboot on all starting modes: console and graphical. This should
avoid that users who need speech or braille land in an inaccessible
graphical environments - unless they chose it, of course.
At end of installation all data that could help investigate are stored
in a compressed archive named INSTALLER.xz, that can be attached to an
email to help investigating issues like no Internet connection or no
sound or speech. These data include the file syslog where all messages 
from the kernel or userspace applications relying on syslogd are written.
This also free /dev/tty4, where these messages were previously sent so
users have one more tty at hand.
